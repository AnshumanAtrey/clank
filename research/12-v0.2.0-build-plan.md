## North star (verbatim from issue #4)

> A user runs `clank deep <number>` with **zero configuration** — no API keys, no Telegram pair, no WhatsApp QR scan — and gets richer output than they would from PhoneInfoga, ignorant, mr.phone, or any combination of free OSINT tools. They should never need to install a second phone-OSINT tool again.

**Concrete acceptance test (US/EU number, fresh install):** carrier (libphonenumber + OVH cross-validate) · WhatsApp registered + display name (no QR pair) · spam score (community sources) · 2+ name attributions · 2+ social presence hits · dorks URL list inline · total runtime < 5 s.

## What changed since the April 2026 research

Three things invalidated parts of the prior plan:

1. **Truecaller is permanently off-menu.** The Play Integrity attestation rolled out early 2026 is hardware-rooted — no community has cracked it; truecallerjs/truecallerpy issues filed through Apr 2026 confirm onboarding still fails. TechCrunch 2026-04-26 reports the company tightening, not loosening. Server-side scrape requires a real Android handset. Drop from v0.2.0 entirely; replace its caller-name slot with Spy Dialer (US) + tellows (EU) + community scrapes.
2. **Google username-recovery hint is patched dead** (Brutecat, June 2025). No 2026 replacement bug. Skip.
3. **Anti-bot has shifted from JA3 → JA4.** Old `HelloChrome_120` utls profiles are now uniquely fingerprintable because Chrome 138+ requires `X25519MLKEM768` in `key_share`. Single-target lookups need a current Chrome JA4 profile or they get blocked. **Pick: `bogdanfinn/tls-client` Chrome_146 profile** — pure Go, no cgo, monthly refresh.

## Architecture changes (issue-#4 explicit)

### A. TLS / HTTP transport layer
Replace `net/http` calls with `bogdanfinn/tls-client` Chrome_146 profile across every external scraper. Single shared `internal/transport/` package; per-call cooperative rate limit (3–5 s per host) with exponential backoff on 4xx. Pin the Chrome version; expect quarterly bumps.

```go
// internal/transport/client.go
package transport

import (
	tlsclient "github.com/bogdanfinn/tls-client"
	"github.com/bogdanfinn/tls-client/profiles"
)

func NewChrome146(timeout time.Duration) (tlsclient.HttpClient, error) {
	return tlsclient.NewHttpClient(tlsclient.NewNoopLogger(),
		tlsclient.WithClientProfile(profiles.Chrome_146),
		tlsclient.WithTimeoutSeconds(int(timeout.Seconds())),
		tlsclient.WithCookieJar(tlsclient.NewCookieJar()),
		tlsclient.WithRandomTLSExtensionOrder(),
	)
}
```

### B. `deep` auto-dorks
At end of `deep.Run`, call `dorks.Generate(canonical)` and append a `Dorks []string` field to `Result`. Render inline in the human formatter; include in `--json`. ~10 LOC change in `internal/deep/deep.go`.

### C. Source-skip rendered as inline suggestions
Drop `Skipped` blocks from the report body; collect them into a `suggestions []string` slice rendered as a footer (`💡 Set IPQS_KEY for fraud score`). UI fix in `internal/deep/cmd.go` (~30 LOC).

### D. Locale-aware dispatch
Each source declares `Regions []string` (ISO-2). `deep.Run` consults libphonenumber's region detection and skips sources whose `Regions` doesn't include the detected region. Sources with empty `Regions` always run. ~60 LOC across the source orchestrator + per-source registration.

### E. Per-source disk cache `~/.clank/cache/<source>/<sha256(phone)>.json`
Generic `cache.Wrap(source, ttl)` middleware. Default TTL 10 min; respects the `--no-cache` flag. Bypassed for sources that return time-sensitive signals (HLR live status). ~120 LOC.

### F. Default timeout 5 s
Drop `Options.Timeout` default from 60 s to 5 s. Per-source can request its own (e.g. `tellows` 8 s, `wa.me` 12 s). Fan-out remains `wg.Wait` parallel.

### G. Drop tech debt
Delete `internal/local/data/spam-oros42.csv` and any code paths citing it. Replace MCC/MNC source from `pbakondy/mcc-mnc-list` to `CursedHardware/mno-list` (weekly-refreshed JSON).

## Sequencing — three minimum-viable releases

### v0.1.1 — architecture-only (1 day)
Items B + C + F (auto-dorks, inline suggestions, 5-s timeout). No new sources. Zero risk.

**What changes for the user:** `clank deep` ends with ~30 actionable Google/Bing dork URLs and a 5-s ceiling. Acceptance score moves from "fail at 2 signals" to "fail at 4 signals" thanks to the dork list alone.

### v0.2.0-rc1 — zero-config blitz (1–2 weeks)
Tier-A items 1–11 + architecture A (TLS layer) + D (locale dispatch) + E (cache) + G (drop tech debt).

**What changes for the user:** clean install on a US/EU number returns ≥6 sources of evidence in <5 s (carrier + WhatsApp + CNAM + FCC + FTC + Wayback + GitHub commits + tellows + Spy Dialer + Microsoft hint), beats PhoneInfoga zero-config baseline. Issue #4 acceptance bar passes for US/EU.

Tier-B items 12–21.

**What changes for the user:** Indian / Brazilian / French / Japanese / Israeli / Canadian numbers also get rich output; locale-aware dispatch only runs the relevant sources per region. Acceptance bar passes for IN, BR, JP, FR, IL, CA in addition.

### v0.2.0-rc3 — bonus signals + opt-in surfaces (optional, 1 week)
Tier-C items 22–27 + opt-in flag wiring.

## Implementation contract (per-source adapter shape)

Every new source implements:

```go
// internal/sources/source.go
package sources

import "context"

type Source interface {
	Name() string                  // "wame", "ovh", "fcc"
	Regions() []string             // ISO-2 list; empty = global
	Lookup(ctx context.Context, e164 string) (Result, error)
}

type Result struct {
	Source   string         `json:"source"`
	Found    bool           `json:"found"`
	Fields   map[string]any `json:"fields,omitempty"`
	URLs     []string       `json:"urls,omitempty"`
	Took     string         `json:"took"`
	Skipped  string         `json:"skipped,omitempty"`
	Error    string         `json:"error,omitempty"`
}
```

`deep.Run` becomes a registry walk over `[]Source`, filtered by region match, fanned out concurrently, results aggregated by `Source.Name()`. The current 6 hardcoded blocks collapse into the same registry.

## Things explicitly NOT changing in v0.2.0

- No headless-browser dependency in the default install. `--browser` flag gates rod for the rare HTML-drift cases (Microsoft JS path, wa.me fallback).
- No paid CAPTCHA solver. clank stays single-target friendly; sleep-past, don't solve.
- No new auth-required sources in default. Anything needing a key stays opt-in.
- No mobile-app-gated sources (Truecaller, Sync.me, Eyecon class).
- No SS7 / SigPloit / attack tooling.
